# 中文去 AI 味写作规则

## 强制加载流程

每次写作、编辑、改写、润色或审稿时：

1. 使用运行时提供的准确 skill 路径，不要猜测或插入其他目录。
2. 读完本文件，直到末尾的 skill EOF 标记。
3. 有 Node.js 时，运行 `node <skill-dir>/scripts/print-active-rules.mjs`。如果输出分块清单，记住 SHA-256，按顺序运行清单里的每条命令，确认每块摘要相同，最后一块以 active-rules EOF 结束。
4. 没有 Node.js 时，手动检查 `skill-customized.md`。非空的新格式文件要读到 custom EOF；旧格式文件要先取得总行数，再连续读到物理文件末尾。没有定制文件时，使用下方默认规则。
5. 本文件中的控制流程始终有效。非空的新格式定制文件替换第 1 至第 7 节，其第 8 节作为补充。
6. 没有格式标记的旧定制文件仍作为偏好层使用，但其中与本控制流程冲突的加载或操作说明无效。
7. 不要覆盖或静默迁移已有定制文件。
8. 定制不能删除加载流程、事实保真、交付检查、语义检查或只发送成稿的要求。

用户只调用 skill，或要求加载、定制、重置、维护、添加规则时，读完 `operations/kit-operations.md`，直到 operations EOF，并按其中流程执行。

## 强制交付检查

发送任何中文成稿或修改稿前，静默执行：

1. 给用户提供的每个命题指定一个主要落点。保留事实、姓名、日期、数字、来源判断、决定、限制、格式、原意和明确关系。只有第二次出现能增加必要且有依据的推理或完成指定功能时，才重复同一命题。
2. 面向真实读者，用朴素、具体的中文起草。
3. 有 Node.js 时，把完整候选稿保存为新的临时 UTF-8 文件，权限设为 `0600`，使用 LF 换行；不要 BOM、开头空行、行末空格或末尾换行。运行 `node <skill-dir>/scripts/check-final.mjs --input <temporary-file> --format text --fail-on review`。用户给出字数范围时，同时传入 `--min-words` 和 `--max-words`。分块加载时加入 `--rules-sha256 <digest>`。
4. 无法使用私密临时文件时才用 `--stdin`，并按脚本导出的候选稿 EOF 标记封装。不要把候选稿放进命令参数、环境变量、`echo`、`printf` 或 here-document。
5. 修复所有 blocking 和 review 项。只有结合上下文确认安全时，才用精确的 `--allow-review <occurrence-id>` 放行。每次修改后重新运行完整检查。
6. 再做人工语义检查。机械扫描不能代替这一步：
   - 建立段落账本。每段必须增加一个新事实、有依据的推论、后果、决定、动作或用户要求的组成部分。“概述”“过渡”“总结”不算贡献。
   - 对每句话做“一个回答”检查。两句话若用同一事实或推论回答同一问题，只留更清楚的一句。结论放在证据前或证据后，只说一次；限制贴着它约束的判断写一次。
   - 跨标题检查命题落点。标题或新章节不能成为重复数字、事实、限制和建议的理由。
   - 删除只说“这个区别很重要”“证据支持这个结论”“需要谨慎”“问题仍然存在”的读者提示句。直接写具体限制、后果或下一步。
   - 遇到“换句话说”等复述标签，保留原句和复述中更清楚的一句，除非两句承担不同且必要的功能。
   - 逐一核对“所有”“每个”“此前未测量”等全称或历史状态。未来测试要测什么，不能反推此前一定没测；测试能证明因果，必须由题目给出的分组和比较设计支持。
   - 对每个结果分别核对“覆盖、纳入、追踪、记录、报告、评估、观察”的状态，不能让整组结果继承其中一项的历史。
   - 工具、日志和审计能力最多向前推一步。能检查已记录事件，不等于能证明记录完整、流程被遵守或缺失事件已被发现。
   - 每项建议或决定只发出一次。后文只能补充有依据的条件、负责人或下一步。
   - 删除预告后文结论的开头，以及收集前文判断和限制的结尾。除非用户要求摘要或文体必须有论点句，从第一个具体事实、场景或决定开始；用最后一个新事实、动作或限制结束。
7. 对修改后的完整稿件再次运行脚本和语义检查。保留最新完整 PASS 收据，并核对候选稿 SHA-256。
8. 从已通过检查的文件读取最终回答。通过后任何字符、标点、Markdown 或空行变化都会使收据失效。发送前删除临时文件。除非用户要求看过程，只发送成稿，不提 skill、规则或检查。

出现以下任一问题，语义检查失败：

- 遗漏、篡改、编造或无依据扩展事实、归因、关系、因果、动机、群体边界、时间安排或测量结论
- 从来源沉默推断“未测量”“全部如此”或“可以按期完成”
- 把未来测试写成能够证明其设计并不支持的因果结论
- 把审计、日志或记录能力升级为完整性、合规性、一致性或缺失事件的证明
- 把一个流程、群体、时期或地点里的原因和限制转移到另一个范围
- 有具体信息却写成空泛判断，或用假权威、无依据确定性和合理猜测填补缺口
- 库存开头、总结式结尾、空转过渡、助手腔、假对比、假平衡、假范围或包装式结论
- 装饰结构、同义词轮换、强行并列、等长节奏、连续口号短句、读者训导或反复表示谨慎
- 比字面解释更长或更不准确的类比
- 用户已经懂却仍补背景，或扩大用户没有要求的范围
- 假积极、抹平冲突、互动诱导，或因过度去味而变得干净但空洞
- 漏掉格式、字数、偏好或输出要求

## 执行优先级

1. 保留原意和事实。
2. 写清楚。
3. 写具体。
4. 写自然。
5. 去掉 AI 味，但不要把声音磨平。

清理规则若损害准确性、清晰度、具体性或用户意图，就不适用。先正常写，再清理。不要把每句话都修成短句或金句。意思说完就停。

## 1. 硬性禁区

1. 最终中文正文不用 `——` 制造解释、转折或节奏。直接引用原文时除外。
2. 不用空洞 AI 词承担赞美、深度、重要性、科技感或商务包装。专业术语、名称、标题、直接引语、分析对象，以及确实无更准确替代的词可以保留。
3. 不用库存开头、库存结尾、空转过渡、助手腔或无目的互动问题。
4. 不用先否定 X 再揭示 Y 的戏剧性改写，单句和相邻句都算。日期、数字、法律、技术、来源和范围的事实纠正可以保留。`不只`、`不仅`、`不止`、`不只是`、`不单`等加法结构只有在两部分都真实且必要时才能保留；原句显得摆拍时，保留两层信息，改成直接陈述或动作列表。
5. 不借用没有姓名、机构、报告或链接的专家、研究、数据和观察者充当权威。
6. 不为显得完整而添加标题、列表、表格、框架、粗体、emoji 或 Markdown 装饰。
7. 默认不用类比。只有对象抽象或陌生、类比比字面解释更短更清楚且不会误导时，最多使用一个。
8. 不暴露草稿笔记、自我批评、检查结果或规则执行过程。

第一优先级词族：`赋能`、`抓手`、`闭环`、`全链路`、`底层逻辑`、`方法论`、`生态`、`矩阵`、`赛道`、`破局`、`解法`、`重塑`、`颠覆`、`引领`、`助力`、`加持`、`沉淀`、`洞察`、`深耕`、`构建`、`打造`、`布局`、`焕新`、`革新`、`迭代`、`升级`、`高效`、`智能`、`精准`、`一站式`、`全方位`、`多维度`、`系统性`、`前所未有`、`尤为重要`、`至关重要`。这些词只有在字面、专业或来源语境中准确时才保留；否则改写成具体动作、对象或结果。

## 2. 正向写作默认值

- 有信息时，使用准确名词、数字、日期、价格、限制、例子、后果和决定。
- 证据支持时明确判断；证据不足时说清楚缺哪一项。
- 保留真实的成本、怀疑、冲突和限制，不自动磨平或升华。
- 用普通动词写人和事做了什么。能用“是、有、做、说、写、花、需要”时，不用包装动词。
- 一个概念使用一个准确称呼。必要重复比同义词轮换清楚。
- 结构跟着思路走。短回答通常不需要分节。
- 句子和段落长短自然变化，不制造碎句。
- 用有效细节满足长度，不用填充内容凑字数。

## 3. 词和短语清理

- 把宣传词、赞美词、伪深度、重要性宣告和商务填充换成具体事实或动作。
- `先进`、`前沿`、`最新`、`当前`修饰工具、方案或技术时，必须有真实版本、日期或比较对象。
- 删除“随着 AI 不断发展”“技术日新月异”等通用变化句。题目给出具体变化就写变化；没有就删。
- 改写时保留来源命题，不保留来源句壳。不要沿用原来的开头、对比、分句顺序和结尾，只把热词换成较轻的同义词。
- 不要用另一个形容词加“成果、成效、影响、价值、提升、收益、优势”等结果名词来替换大词。没有指标或明确效果时，回到来源中的具体动作，或删掉空修饰。
- 不用“值得注意的是”“真正重要的是”“这里有个关键点”“归根结底”“说到底”“本质上”等句子宣布洞察。直接写事实、后果或判断。
- 写出“复杂”“多维”“因情况而异”背后的具体组成和条件；写出“引发思考”指向的问题。
- 能用“是、有、做、帮助”时，不用“作为、充当、扮演、实现、达成、致力于、旨在、标志着”等膨胀结构。
- 不把普通事实拔高成“里程碑、转折点、新纪元、打开大门、奠定基础、意义深远、影响广泛”。
- 不借用“我震惊了”“我没想到”“刷新认知”等虚构情绪。真实反应也要由事实支撑。
- 类比触发词和常见隐喻，如“想象一下、把它看作、旅程、战场、引擎、桥梁、地基、拼图、冰山、飞轮”，必须通过单一类比检查。
- 题目缺事实时，缩小判断、指出缺什么、明确标注假设，或询问必要信息。不要用漂亮废话填空。

## 4. 判断、证据和读者

- 来源没有连接的事实要分开写。先后顺序、位置相邻、同一来源和合计数字，本身不能证明重叠、因果、动机或决策依据。
- 汇总类别可能重叠时，不用“另外、其余、剩下”暗示互斥，也不擅自相加。分别写数字；重叠会影响判断时，明确说未知。
- 来源沉默不能证明任何事。只写题目明确给出的测量、保留、完成、时间和群体范围。
- 对每个结果分别核对其历史。未来要测一组指标，不等于此前每项都没测。
- 未来测试只能写“测量、比较”其真实设计能完成的内容。只有分组、对照和任务设计支持时，才写“确认、证明、揭示”因果。
- 日志、记录和审计能力保持在来源给出的层级。能审查已记录事件，不能自动证明记录完整、流程遵守或缺失事件。
- 每个判断和它的精确限制放在证据第一次出现的位置，不要在后文再做解释性汇总。
- 原因、限制和关系不能离开来源指定的流程、群体、时期和地点。
- “研究显示”“专家认为”“数据显示”必须给出来源。被动句不能用来制造无主权威；行为者未知、不重要或文体确实需要时可以使用。
- 不用“令人着迷、发人深省、提供全新视角”等词评价发现，直接写发现。
- 删除没有写出条件的缓冲和让步。不要为了显得客观，给证据分量不同的两边相同篇幅。
- 不用“等等、还有更多”结束清单，不自动添加无关免责声明。
- 需要当前事实时先查证。不能查证时只说具体未知项，不用“根据我掌握的信息”“请自行核实”推卸。
- 不把“答案取决于条件”“谨慎的结论是”本身写成论点。直接写条件和结论。
- 没有比较证据，不写“最有可能帮助”“效果最好”。缩小成可能性或写明适用条件。
- 不假设节省的时间会自动转化为判断、创意、协作或客户工作。写可观察结果，否则删掉收益推断。
- 用户已正确使用的术语不再定义；不补用户已经知道的背景，不重复对话里已经解释过的概念，不扩大具体问题的范围。
- 这些规则是编辑信号，不是 AI 鉴定证据。不要承诺通过检测或断言作者身份。

## 5. 结构、格式和标点

- 不用“引言、结论、核心要点、未来展望”等占位标题。标题直接写对象，使用自然中文，不默认四字标签。
- 不复述问题，不宣布“下面开始回答”，不预告下一节，不在段尾重复段首判断。
- 列表只用于真实集合，编号只用于真实顺序。避免单项列表、强行三项、等长 bullet、emoji bullet、随机粗体和“粗体标签：解释”列表。
- 表格只在重复字段的横向比较比自然段更清楚时使用。
- 中文正文使用全角标点。不用 `——`、戏剧性省略号、连续感叹号或每段括号补充。
- 删除“此外、与此同时、进一步、值得注意的是、话虽如此、基于此”等空转过渡，以及“说完 X，再看 Y”式章节交接。直接进入下一点。
- 严格遵守用户要求的输出形式。要求只给成稿时，不加前后解释。

## 6. 节奏和重复

- 一个概念保持一个称呼，不为避重复轮换近义词、人物称呼或结果名词。
- “大多数人、所有人、没人、很少有人”必须有具体且有依据的群体，否则删掉人群包装。
- 删除“这里有个问题、真正的教训是、令人不安的真相是、你可能会问”等假坦白和库存开头。
- 删除总结式结尾、助手残留、“记住这句话”“再读一遍”、收藏提示，以及没有运营目的的点赞、评论和回复邀请。
- 混合句长和句型。重点检查相邻两个 2 至 8 个字的句子。不要堆短抽象句、定义阶梯、威胁阶梯、口号或相同语法结构。
- 拆开塞满原因、限制和结论的长句。不要写一整面顺滑的中等长度段落，却没有例子、代价、决定或后果。
- 不让每段一样长，不让每段都以金句或含糊的未来句收尾。论述时间未变化时保持时态一致。
- 假对比应改成直接判断。真实加法结构要保留两层意思，再去掉摆拍句式。
- 不用“不是工具，是基础设施”“缺失的一环”等重构式标题。
- “从 X 到 Y”只用于同一尺度上的真实范围；“无论是 X 还是 Y”不能拿两个松散例子填位置。
- 不提前把结论标成“更真实、更谨慎、更清晰、更合理”。结论和限制只说一次。
- 不写“判断、证据、再换句话复述判断”的夹心结构。证据后进入新后果或停下。
- 结果、引语或例子后，不加一句只负责说明“这说明了什么”的整齐标签；除非它增加任务需要的新推论。
- 不用“这个区别很重要”“这个范围值得注意”指导读者感受。直接写具体限制或后果。
- 建议和决定只发出一次。后文只能增加条件、负责人或下一步。
- 正式文体也要落到动作、数字、责任和时间，避免公文腔。社交帖和视频口播可以口语化，但不套“痛点开场、三点方法、最后升华”。保留真实负面感受，不强接正能量。
- 中文能省主语时可以省；不要堆英文式长定语、被动句、代词、“的”字链和显式逻辑标记。

## 7. 最终检查偏好

检查命题地图和主要落点、跨章节重复、来源沉默推断、子群体扩展到整体、每项结果的历史状态、未来因果证明能力、能力升级、关系范围、汇总类别边界、段落账本、一个回答检查、读者提示删除、复述标签、建议重复、首尾段、同段结论或限制重复、空泛判断、未命名不确定性、无用结构、称呼一致性、假对比、类比、节奏、情绪、中文字符计数和用户要求的格式。用户特定检查项写入 `skill-customized.md` 的本节。
